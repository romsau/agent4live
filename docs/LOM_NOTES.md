# LOM — Conventions & Limites

Conventions transverses des outils MCP et catalogue de ce qui n'est volontairement **pas** exposé. Pour la liste des outils existants, lire `app/server/tools/<famille>.js` (source de vérité, toujours à jour) ou `tools/list` du serveur MCP en marche.

**État actuel : 232 outils répartis en 14 familles** (`raw`, `session`, `transport`, `tracks`, `clips`, `scenes`, `arrangement`, `application`, `racks`, `instruments`, `browser`, `tuning`, `midi`, `meta`).

---

## Conventions communes

- **Indices** : 0-based partout (`track=0` = première piste, `slot=0` = premier clip slot, `index=0` = première scène).
- **Path LOM** : pour `lom_get` / `lom_set` / `lom_call`, le path est space-separated (ex. `"live_set tracks 0 mixer_device volume"`).
- **Read-back recommandé** : après une écriture, lire la valeur via `lom_get` et observer Live visuellement pour confirmer que le changement a pris. Les méthodes LOM peuvent silencieusement no-op sur mauvais format ou propriété read-only.
- **Couleurs** : passées en int RGB 24-bit (`0xRRGGBB`). Live affiche le swatch le plus proche de sa palette, donc le read-back peut différer du `value` envoyé.
- **Range / validation** : les schémas Zod valident côté serveur (ex. `value` entre 0 et 1 pour volume, 0-127 pour MIDI velocity). Out-of-range = rejet au niveau MCP, pas d'appel LOM inutile.

---

## Limites — pas d'API LOM publique

Live n'expose pas de méthode pour ces actions. Pas de workaround côté MCP — il faudrait du UI scripting hors-LOM (qui n'est pas dans le scope d'agent4live).

| Action                                                                | Raison                                                                                                                                                                                                                                                                                                                                        |
| --------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Freeze / unfreeze track                                               | `Track.is_frozen` est read-only. Pas de méthode publique pour déclencher le freeze.                                                                                                                                                                                                                                                           |
| Copier-coller des clips entre slots non-adjacents                     | Pas de `Clip.copy` / `paste` ; passer par `duplicate_clip_to` (slot adjacent) ou `add_clip` + read/write notes.                                                                                                                                                                                                                               |
| Latence < 10 ms (live performance scheduling)                         | LOM round-trip = ~10-50 ms minimum. Pour du temps-réel précis, utiliser MIDI clock ou Ableton Link, pas la LOM.                                                                                                                                                                                                                               |
| Renommer le master track                                              | `Track.name` est documenté settable sans note master-only mais Live 12 ignore silencieusement le SET sur master. Le rename UI right-click marche. Pas exposé comme outil pour éviter de prétendre que ça fonctionne.                                                                                                                          |
| Toggle Song.exclusive_arm / exclusive_solo via LOM                    | Documentés `bool` get/set mais en pratique Live les traite comme une simple miroir de la Pref (Live's Preferences > Record/Warp/Launch). LOM SET silently ignored. À toggler via la fenêtre Preferences manuellement.                                                                                                                         |
| Track monitoring state (Off/Auto/In)                                  | Pas de `monitoring_state` ni `current_monitoring_state` dans le corpus LOM Live 12.3.5 (vérifié par `grep -r monitor` sur les classes LOM). Le contrôle UI existe par track header, mais sans accès LOM exposé.                                                                                                                               |
| Supprimer une take lane                                               | Pas de `Track.delete_take_lane` dans la LOM Live 12.3.5. Création + rename OK ; suppression doit se faire manuellement dans Live (right-click).                                                                                                                                                                                               |
| Toggle Song.tempo_follower_enabled (Live 12)                          | Documenté `bool` mais explicitement marqué **read-only** dans la LOM. Lecture OK via lom_get, mais setter impossible. À toggler via la transport bar manuellement.                                                                                                                                                                            |
| Effacer Clip.groove (clear assigned groove)                           | Set `Clip.groove = id 0` est silencieusement ignoré. Swap entre grooves OK, mais retour à "None" doit se faire via UI Live (Clip > Groove dropdown > None).                                                                                                                                                                                   |
| Set `Device.is_active` directement                                    | `Device.is_active` est une propriété dérivée read-only (= 0 si le device OU son rack parent est off). Pour toggler le device, il faut set `parameters[0].value` (le "Device On" param). C'est ce que fait `set_device_active`.                                                                                                                |
| Set DrumPad.name/note/mute/solo sur pads vides                        | Les pads sans chains/samples ignorent silencieusement les SET. Empiriquement vérifié : `set name` sur drum_pads[36] (vide) reste "C1". Sur pads peuplés (kit chargé) les sets semblent prendre. La doc LOM ne documente pas cette restriction. Toujours read-back.                                                                            |
| `DrumPad.note` setter                                                 | Particulièrement étrange : drum_pads est indexé par MIDI note (drum_pads[36] = pad triggered par note 36). Le setter `note` est probablement 100% read-only pour cette raison structurelle.                                                                                                                                                   |
| `CuePoint.time` setter                                                | Documenté `float` (pas read-only) mais Live 12 ignore silencieusement le SET (vérifié empiriquement, même via résolution par id). Pour déplacer un cue : delete à l'ancienne position (set_or_delete_cue) puis recreate à la nouvelle.                                                                                                        |
| Path `live_set cue_points N` (et autres list-of-LOM-objects)          | Le path par index ne résout pas comme attendu via `new LiveAPI(null, 'live_set cue_points N')` — id retourné = 0. Pour adresser une CuePoint il faut résoudre son id via `song.get('cue_points')` puis instancier `new LiveAPI(null, 'id ' + N)`. C'est ce que font tous nos handlers Case B (cue points, drum pad chains, devices id, etc.). |
| Toggle `Song.back_to_arranger` / `Track.back_to_arranger` directement | Documentés `bool` get/set/observe mais Live ignore silencieusement le SET (état dérivé du playback : "des session clips override-t-ils l'arrangement ?"). Pour repasser à l'arrangement, utiliser `back_to_arranger()` (la méthode qui clear l'override).                                                                                     |
